# v3.20 代码审查 — 验收标准

## AC-1: 鸭子纬度改变时底部接触与垂直校正

**根因分析**：
`placeDuck()` 中鸭子位置使用硬编码 `1.027` 作为距球心距离。鸭子模型身体（SphereGeometry r=0.038, Y-scale=0.72）的半高为 0.02736，原点需在 `1.0 + 0.02736 = 1.02736` 处才能让底部刚好接触地球表面（r=1.0）。误差虽小，但对建模精度有影响。

**验收条件**：
- [ ] `placeDuck()` 中鸭子距球心距离由模型几何参数显式计算（`1.0 + bodyRadius * bodyScaleY`），不再使用硬编码 `1.027`
- [ ] 鸭子局部 Y 轴始终沿球面法线方向（向外），即 `makeBasis(east, up, north)` 中 `up` 为表面法向量
- [ ] 调节鸭子纬度（-90° 到 90°）时，鸭子始终站立在地球表面，底部接触表面、头部朝外
- [ ] 调节地球自转时鸭子跟随球面旋转，不陷入球内、不浮空

## AC-2: 点击时区按钮后显示白色方框

**根因分析**：
`createTZGroup()` 的 `makeTex()` 函数创建 Canvas 后直接 `fillText` 绘制文字，未显式清空 Canvas 背景。Android WebView 中 Canvas 初始像素可能非透明，导致 Sprite 纹理背景渲染为白色矩形。

**验收条件**：
- [ ] `makeTex()` 中在绘制文字前调用 `cx.clearRect(0, 0, cv.width, cv.height)` 清空 Canvas 为透明
- [ ] `createSunPathLabels()` 中同理加 `clearRect`（同类问题，预防性修复）
- [ ] 点击时区按钮后，时区标签（+8, +7 等）仅显示彩色文字，无白色矩形背景
- [ ] 太阳轨迹时间标签（0:00/6:00/12:00/18:00/24:00）无白色矩形背景
