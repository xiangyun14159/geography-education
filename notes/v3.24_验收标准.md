# v3.24 验收标准

## AC-1: 地球纹理正常渲染（非黑球）

**根因分析**：
1. v3.23 引入 `<img id="texImg" src="earth_tex.png">` 试图加载 2048x1024 PNG，Android WebView 中 `file://` 访问受限且大纹理可能超出设备 `MAX_TEXTURE_SIZE`，导致纹理加载失败 → 黑球
2. `proceduralEarthTex()` 降级函数被错误修改：海洋从深蓝改为奶油色后，`isLand` 检测逻辑不匹配，且 `earthMat` 初始 `color:0x3388cc` 无纹理时渲染为暗色

**修复方案**：
- 移除 `<img>` 标签加载 PNG 的方案（Android WebView 不可靠）
- 直接使用 `proceduralEarthTex()` 生成 Canvas 纹理（v3.19 验证过的可靠方案）
- 将海洋颜色从深蓝改为浅蓝（保持蓝调便于陆地检测），陆地颜色调亮为粉彩风格

**验收条件**：
- [x] 页面加载后地球显示为浅蓝色海洋 + 粉彩大陆的纹理球体，非黑球/暗球
- [x] 不依赖外部 PNG 文件，纯 Canvas 生成纹理
- [x] 纹理在 Android WebView 中正常渲染（Canvas 纹理 + SRGBColorSpace）
- [x] 大陆轮廓（非洲、欧亚、北美、南美、澳洲等）清晰可辨
- [x] 鸭子影子在地表可见（浅蓝海洋 #c8daea~#d2e1f0 + 浅绿陆地 #c5cf9e）
