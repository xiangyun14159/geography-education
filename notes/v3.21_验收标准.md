# v3.21 代码审查 — 验收标准

## AC-1: 鸭子纬度改变时底部接触与垂直校正

**根因**：`placeDuck()` 使用硬编码 `1.027` 作为鸭子距球心距离，该值未与模型几何参数关联，且 `updateSunPath()` 和 `updateSunPathLabels()` 中也有同样的魔数，三处不一致时会造成轨迹与鸭子位置不匹配。

**修复**：
- `index.html:619-620` — 新增 `_duckBodyR`, `_duckBodySY`, `_duckHalfH`, `_duckDist` 四个变量，由鸭子模型几何参数 `SphereGeometry(0.038) * scaleY(0.72)` 显式计算 `_duckDist = 1.0 + 0.038*0.72 = 1.02736`
- `index.html:625` — `placeDuck()` 使用 `_duckDist` 替代 `1.027`
- `index.html:661` — `updateSunPathLabels()` 使用 `_duckDist` 替代 `1.027`
- `index.html:715` — `updateSunPath()` 使用 `_duckDist` 替代 `1.027`
- 全局搜索确认代码中已无 `1.027` 魔数残留

**验收条件**：
- [x] `placeDuck()` 中鸭子距球心距离由模型几何参数显式计算（`1.0 + bodyRadius * bodyScaleY`），不再使用硬编码 `1.027`
- [x] 鸭子局部 Y 轴始终沿球面法线方向（向外），即 `makeBasis(east, up, north)` 中 `up` 为表面法向量（未修改，原本正确）
- [x] `updateSunPath()` 和 `updateSunPathLabels()` 同步使用 `_duckDist`，轨迹与鸭子位置一致
- [x] 调节地球自转时鸭子跟随球面旋转，不陷入球内、不浮空 ✅

## AC-2: 点击时区按钮后显示白色方框

**根因**：`createTZGroup()` 的 `makeTex()` 和 `createSunPathLabels()` 中，Canvas 创建后未显式清空背景。Android WebView 中 Canvas 初始像素可能非透明（取决于 WebView 实现），导致纹理渲染为带白色/黑色背景的矩形。

**修复**：
- `index.html:571` — `makeTex()` 中 `fillText` 前增加 `cx.clearRect(0,0,cv.width,cv.height)`
- `index.html:638` — `createSunPathLabels()` 中 `fillText` 前增加 `ctx.clearRect(0,0,cv.width,cv.height)`

**验收条件**：
- [x] `makeTex()` 中在绘制文字前调用 `cx.clearRect(0, 0, cv.width, cv.height)` 清空 Canvas 为透明
- [x] `createSunPathLabels()` 中同理加 `clearRect`（同类问题，预防性修复）
- [x] 点击时区按钮后，时区标签（+8, +7 等）仅显示彩色文字，无白色矩形背景 ✅
- [x] 太阳轨迹时间标签（0:00/6:00/12:00/18:00/24:00）无白色矩形背景 ✅
