## 一、高中地理太阳视运动轨迹核心知识点

### 1.1 基本概念

| 概念 | 符号 | 说明 |
|------|------|------|
| 观测者纬度 | φ | -90°~+90°，北正南负 |
| 太阳赤纬 | δ | -23.44°~+23.44°，随日期变化 |
| 时角 | H | (地方时 - 12) × 15°/h，正午 H=0，上午负、下午正 |
| 太阳高度角 | h | 太阳与地平面的夹角，0°=地平线，90°=天顶 |
| 太阳方位角 | A | 太阳在地平面上的投影方向 |

### 1.2 核心公式

**高度角公式（天球坐标系基本方程）：**

```
sin(h) = sin(φ)·sin(δ) + cos(φ)·cos(δ)·cos(H)
```

**方位角公式（从北顺时针，中国高中教材惯例）：**

```
sin(A) = -cos(δ)·sin(H) / cos(h)
cos(A) = [sin(δ) - sin(φ)·sin(h)] / [cos(φ)·cos(h)]
```

**正午太阳高度角（H=0 时）：**

```
h_noon = 90° - |φ - δ|
// 等价形式（北半球）：h_noon = 90° - φ + δ
```

**日出时角（h=0 时）：**

```
cos(H_rise) = -tan(φ)·tan(δ)
```

**昼长：**

```
昼长 = 2 × acos(-tan(φ)·tan(δ)) / 15   （单位：小时）
```

### 1.3 不同纬度的典型视运动规律

#### 赤道（φ=0°）
- 太阳始终从**正东升、正西落**（全年不变）
- 正午太阳在**头顶正上方**（天顶）或附近
- 昼夜**永远等长**（12h）
- 太阳视运动轨迹**对称**于东西轴
- 二分日：太阳经过天顶

#### 北回归线（φ=23.44°N）
- **夏至**正午太阳在**天顶**（90°）
- 全年正午太阳在**南方天空**
- 夏至日出**东北**、日落**西北**；昼最长
- 冬至日出**东南**、日落**西南**；昼最短

#### 北极圈（φ=66.56°N）
- **夏至**：极昼（24h白天），午夜太阳擦地平线
- **冬至**：极夜（24h黑夜），正午太阳擦地平线
- 春秋分：正常昼夜交替

#### 中纬度（以北京 φ=40°N 为例）
- 全年正午太阳在**正南**方向
- **夏至**：日出东北、日落西北，昼长约 15h，正午高度角约 73.5°
- **冬至**：日出东南、日落西南，昼长约 9h，正午高度角约 26.5°
- **春/秋分**：日出正东、日落正西，昼长 12h，正午高度角 = 90°-φ = 50°

#### 北极点（φ=90°N）
- 太阳高度角 = 赤纬值（恒等于 δ）
- 太阳沿**等高线**绕圈运动（不升不落）
- 春分→秋分：极昼（太阳始终在地平线上方绕圈）
- 秋分→春分：极夜

### 1.4 太阳视运动轨迹图的"三线"特征（高中地理核心图）

对于北半球中纬度观测者，太阳视运动轨迹图通常显示三条路径：

```
         天顶(Z)
           |
    夏至  / \  昼最长、轨迹最高
    ----/-----\---- 地平线
   春分/       \秋分  昼夜等长、轨迹居中
    --/---------\-- 地平线
   冬至          冬至  昼最短、轨迹最低
   /             \
  N(北)    E(东)   S(南)   W(西)  N(北)
```

| 日期 | δ | 日出方向 | 正午方向 | 日落方向 | 轨迹特点 |
|------|---|---------|---------|---------|---------|
| 夏至 6/21 | +23.44° | 东北 | 正南（天顶以南） | 西北 | 最高、最长 |
| 春/秋分 | 0° | 正东 | 正南 | 正西 | 居中 |
| 冬至 12/21 | -23.44° | 东南 | 正南（最低） | 西南 | 最低、最短 |

**南半球中纬度**：正午太阳在**正北**方向，夏至（12月）和冬至（6月）互换。

### 1.5 极昼/极夜条件

| 条件 | 判断 |
|------|------|
| 极昼 | cos(H_rise) ≤ -1，即 -tan(φ)·tan(δ) ≤ -1 |
| 极夜 | cos(H_rise) ≥ 1，即 -tan(φ)·tan(δ) ≥ 1 |
| 正常 | -1 < cos(H_rise) < 1 |

等效判断（北半球）：
- 极昼：φ > 90° - |δ|（且 φ 与 δ 同号）
- 极夜：φ > 90° - |δ|（且 φ 与 δ 异号）

### 1.6 正午太阳高度角年变化

对于固定纬度 φ：
- 冬至最低、夏至最高（北半球 φ>23.44°N）
- 在回归线之间（|φ| < 23.44°）：太阳一年两次过天顶
- 年变化幅度 = 2 × 23.44° = 46.88°（回归线以外地区）

## 二、当前代码中的错误分析

### 2.1 当前实现（index.html 第一场景中的 sunPathLine）

当前 `updateSunPath()` (L691-726) 将太阳视运动轨迹画成一个**贴在 3D 地球表面鸭子位置附近的小圆圈**。

**核心问题**：

1. **坐标系统混淆**：把"观测者地平坐标系"(altitude/azimuth → local tangent plane)的曲线硬塞进 3D 地球世界坐标。太阳在天球上的视运动被错误地表示为一个悬浮在地球表面附近的 3D 环。

2. **不符合教学认知**：高中地理教材中，太阳视运动轨迹是**以观测者为中心的半球形天穹图**，不是地球旁边的小曲线。学生看到的是"天球上太阳划过的弧线"。

3. **不能同时展示多条轨迹**：教学中需要同时看到夏至/春秋分/冬至三条轨迹来对比。当前只能显示一条。

4. **没有清晰的地平线参考**：观测者需要看到 N/E/S/W 方位和地平线，才能理解日出日落方位。

5. **高度角/方位角不直观**：需要读数字面板才知道太阳高度，视觉上无法直接感知。

### 2.2 具体数学/代码错误

仔细审查 `updateSunPath()` 的转换逻辑：

```javascript
// 当前代码：在鸭子位置的局部切平面上计算 lx, ly, lz
// lx = cosAlt * sinAz   (东分量)
// ly = sinAlt           (上分量)
// lz = cosAlt * cosAz   (北分量)
// 然后乘以 dist=0.35 再平移到地球表面鸭子位置
```

**这个局部计算本身数学上是正确的**（在局部切平面坐标系中）。**问题不在于公式错，而在于呈现方式错了**——太阳视运动轨迹应该是一个独立的"天球场景"，而不是地球表面的装饰环。

## 三、第二界面正确实现方案

### 3.1 设计目标

单独一个 3D 场景，展示**以观测者（鸭子）为中心的天球视运动**：

```
             天顶 (Zenith)
              |
        夏至 /|\  (最高弧线)
           / | \
    春分/   |   \秋分  (中间弧线)
      /     |     \
冬至/       |       \冬至 (最低弧线)
   /        |        \
  N ───── 地平线 ───── S
        观测者(鸭)
```

### 3.2 场景元素

| 元素 | 说明 |
|------|------|
| **地平线圈** | 半径 1.5 的大圆，水平放置，代表地平线 |
| **方位标记** | N(北)、S(南)、E(东)、W(北) 四个大字 + NE/SE/SW/NW 小字 |
| **观测者（鸭子）** | 位于圆心，站在地平面上，Y=0 |
| **天顶点标记** | 正上方高处的标记 "天顶" |
| **高度参考圈** | 30°、60° 两个高度圈（可选，可切换） |
| **太阳轨迹线** | 3条彩色弧线：夏至(橙)、春/秋分(黄)、冬至(浅蓝) |
| **当前太阳光点** | 沿当前轨迹移动的发光球 |
| **地面阴影** | 鸭子在地面上的实时阴影（随太阳位置变化） |
| **文字标注** | 日出/日落时间、昼长、正午高度角 |

### 3.3 轨迹线计算方法（正确算法）

```
输入：观测者纬度 φ、太阳赤纬 δ、时角 H（0~24小时 → 0~360°）

对每个时角 H：
  1. 计算高度角：h = arcsin(sinφ·sinδ + cosφ·cosδ·cosH)
  2. 计算方位角（从北顺时针）：
     sinA = -cosδ·sinH / cosh
     cosA = (sinδ - sinφ·sinh) / (cosφ·cosh)
     A = atan2(sinA, cosA)
  3. 转换为 3D 局部坐标（Y轴向上，Z轴向北，X轴向东）：
     如果 h < 0（太阳在地平线以下）：不绘制该点
     如果 h >= 0：
       r = R × cos(h)     // 地平面投影半径
       Y = R × sin(h)     // 高度
       X = r × sin(A)     // 东分量
       Z = r × cos(A)     // 北分量
  4. 将 (X, Y, Z) 作为天球上的点画出轨迹
```

其中 R 是半球半径（建议 1.5~2.0）。

### 3.4 三条默认轨迹（北半球中纬度典型）

| 轨迹 | δ | 颜色 | 特点 |
|------|---|------|------|
| 夏至 | +23.44° | 橙红 #ff6622 | 最长、最高，日出东北/日落西北 |
| 春/秋分 | 0° | 金黄 #ffcc00 | 居中，日出正东/日落正西 |
| 冬至 | -23.44° | 浅蓝 #66aaff | 最短、最低，日出东南/日落西南 |

### 3.5 界面切换机制

- 第一界面：3D 地球全局视图（当前已有的场景）
- 第二界面：天球局部视图（新建场景）
- 切换方式：底部 Tab 按钮或场景切换按钮
- 数据共享：两个界面共享同一组变量（duckLat, duckLng, declination, timeOfDay）

### 3.6 特殊场景处理

| 场景 | 处理方式 |
|------|---------|
| 极昼 | 整条轨迹在地平线上方，无升起/落下点 |
| 极夜 | 整条轨迹在地平线下方，不画线（或画虚线） |
| 赤道 | 轨迹对称，夏至偏北/冬至偏南 |
| 南半球 | 正午太阳在北方，夏至和冬至互换显示 |
| 太阳在地平线以下 | 轨迹段不绘制，太阳光点隐藏 |

### 3.7 交互

- **拖动时刻滑块**：太阳光点沿当前日期的轨迹移动
- **拖动赤纬滑块**：当前日期的轨迹实时变形
- **拖动纬度滑块**：所有三条轨迹 + 鸭子位置实时更新
- **预设按钮**：春分/夏至/秋分/冬至（切换 δ）+ 赤道/回归线/北京/极圈（切换 φ）
- **旋转/缩放**：三指或鼠标拖拽旋转视角，滚轮缩放
