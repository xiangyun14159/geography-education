## 逐项详细分析

### 🔴 差异 #1：缺少"当前赤纬"轨迹线（最严重）

**正确方案**：
用户调整赤纬滑块时，天球上应该有一条**对应当前赤纬值**的太阳运动轨迹，
太阳光点沿这条轨迹移动。

**当前实现**：
```javascript
// updateCelestialSunPath() — 只更新3条固定轨迹
_updateOneTrajectory(sunPathSummer, 23.44);   // 夏至
_updateOneTrajectory(sunPathEquinox, 0);       // 春秋分
_updateOneTrajectory(sunPathWinter, -23.44);   // 冬至

// 太阳光点用可变 declination 计算
function updateCelestialSunDot(){
  var s = _sunAltAz(duckLat, declination, timeOfDay);
  // 光点可能不在任何轨迹线上！
}
```

**后果**：
- 用户在赤纬=10°时进入第二界面，太阳光点悬浮在夏至轨迹和春秋分轨迹之间的空白处
- 无法直观看到"当前日期的太阳运动轨迹"是什么形状
- 学生无法理解"赤纬连续变化 → 轨迹连续变形"

**修复**：添加第4条轨迹线 `sunPathCurrent`，颜色亮黄，随 declination 实时更新。

---

### 🟡 差异 #2：地面底图

**正确方案**：
- 以鸭子经纬度为中心投射的地图纹理（或至少清晰的方位线）
- N/E/S/W/NE/SE/SW/NW 八个方位 LABEL 清晰可读

**当前实现**：
- 纯色暗绿 `#1a2a1a` MeshStandardMaterial
- 8个方位标签通过 `_makeAzimuthLabel()` 以 Sprite 悬浮在地面边缘 — 这个其实存在
- 但没有地面上的方位线/十字线/刻度

**修复**：添加地面十字方位线(N-S线 + E-W线)、方位刻度标记。

---

### 🟡 差异 #3：高度参考圈

**正确方案**：
验收方案(v1.1 AC-2)要求"去除 30°/60° 高度参考圈"。

**当前实现**：
```javascript
domeGroup.add(_makeAltRing(30,0x446688,.2));
domeGroup.add(_makeAltRing(60,0x446688,.2));
```
这两条线仍然存在。

**修复**：移除或改为可切换。

---

### 🟡 差异 #4：地平线以下轨迹

**正确方案**：
高中地理教学中，太阳视运动轨迹通常画出**完整的圆**（天球上的大圆），
其中地平线以上的实线是白昼路径，地平线以下的虚线是夜间路径。
这样学生能看到完整的 24 小时运动。

**当前实现**：
`_updateOneTrajectory` 只计算从日升到日落之间的弧段。轨迹始于地平线、终于地平线。
学生看不到太阳在夜晚去了哪里。

**修复**：将轨迹改为完整圆（0h→24h），地平线以下部分用虚线/半透明。

---

### 🟡 差异 #7/#8：缺少关键标记

**正确方案**：
- 日出点、日落点应有标记（方位+时间标注）
- 正午点应有标记（高度角标注）

**当前实现**：
没有任何标记点。

## 修复计划

### 改 A：添加第4条"当前赤纬"轨迹线
- 新增 `sunPathCurrent` + `sunPathCurrentGlow`
- `updateCelestialSunPath()` 中增加 `_updateOneTrajectory(sunPathCurrent, declination)`
- 颜色：亮黄 `#ffee00`，opacity 略高于固定轨迹

### 改 B：轨迹改为完整 24h 圆
- `_updateOneTrajectory` 改为始终计算 0h→24h 所有点
- 地平线以下部分透明化或改用虚线材质

### 改 C：去除高度参考圈
- 移除 `_makeAltRing(30, ...)` 和 `_makeAltRing(60, ...)`
- 或移到 toggle 控制

### 改 D：添加地面方位线
- N-S 延长线（白色虚线）
- E-W 延长线（白色虚线）  
- 方位刻度小标记

### 改 E：添加日出/日落/正午标记
- 轨迹两端各加一个发光小点标记日出日落方向
- 正午最高点加高度角标注
