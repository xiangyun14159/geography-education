## AC-1: `sunPathLabels` 变量覆盖 Bug（关键）

**问题**：`index.html` 第 231 行 `sunPathLabels = createSunPathLabels()` 因函数无返回值导致 `sunPathLabels` 被赋值为 `undefined`，时间标签精灵永不更新位置。

**验收条件**：
- [x] `createSunPathLabels()` 调用不通过赋值语句执行（直接调用 `createSunPathLabels()`）
- [x] 调用后 `sunPathLabels` 保持为包含 5 个 Sprite 的数组（不为 `undefined`）
- [x] `updateSunPathLabels()` 不再因 `!sunPathLabels` 检查提前返回
- [x] 修改鸭子纬度/经度/赤纬/时刻时，5 个时间标签（0:00/6:00/12:00/18:00/24:00）跟随轨迹曲线更新位置

**审查**：`index.html:230-231` — 改为 `sunPathLabels=[]; createSunPathLabels();`，函数向全局数组 push 5 个 Sprite，不再被覆盖。✅

## AC-2: `duckLngVal` HTML 显示值不一致（UI）

**问题**：第 118 行 `<span id="duckLngVal">116°E</span>` 硬编码值与 slider `value="120"` 不匹配。

**验收条件**：
- [x] HTML 中 `duckLngVal` 的初始文本改为 `120°E`
- [x] 页面加载后无短暂闪烁错误文本

**审查**：`index.html:118` — 改为 `120°E`，与 slider `value="120"` 一致。✅

## AC-3: `setAllowFileAccessFromFileURLs(true)` 不必要权限（安全）

**问题**：`MainActivity.java` 第 14 行允许 `file://` URL 互相访问，本 App 仅加载自有 assets 无需此权限。

**验收条件**：
- [x] `MainActivity.java` 中移除 `wv.getSettings().setAllowFileAccessFromFileURLs(true)` 调用
- [x] 地球纹理 PNG（`earth_tex.png`）加载不受影响（纹理通过 `<img>` 标签加载，不使用 file:// 跨域访问）

**审查**：`MainActivity.java:12-14` — 已移除。App 只加载 `file:///android_asset/index.html`，不跨域访问其他 file:// 资源。✅

## AC-4: `usesCleartextTraffic="true"` 多余配置（安全）

**问题**：`AndroidManifest.xml` 第 13 行允许明文 HTTP 流量，但 App 离线运行不需网络。

**验收条件**：
- [x] `AndroidManifest.xml` 中移除 `android:usesCleartextTraffic="true"` 属性
- [x] APK 可正常构建安装（构建验证待 Gradle 执行）

**审查**：`AndroidManifest.xml:12` — 已移除。App 无需任何网络访问。✅
